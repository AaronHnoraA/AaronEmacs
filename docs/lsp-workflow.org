#+title: Language Server Workflow

* Scope

这份文档描述当前语言服务器栈的运行方式、维护入口和扩展约定。

覆盖内容：

- =eglot= / =lsp-mode= 的路由模型
- =Language Server Hub= 和 =Language Server Doctor=
- 常用运维命令和 session 级调参入口
- 如何为新语言补路由、补 Eglot server 映射
- 如何把这种 Hub/Doctor 模式继续扩到别的模块

相关入口文件：

- =lisp/init-lsp.el=
- =lisp/init-lsp-ops.el=
- =lisp/init-lsp-tools.el=
- =lisp/lang/=

* Design

当前策略不是“全局只用一个 LSP 客户端”，而是：

- 默认编程缓冲区走 =eglot=
- 少数明确声明的 major mode 走 =lsp-mode=

也就是：

- 默认路由：=prog-mode= -> =eglot=
- 显式例外：通过 =my/register-lsp-mode-preference= 把某些 mode 切到 =lsp-mode=
- 本地自定义 Eglot server 映射：通过 =my/register-eglot-server-program= 注册

这样做的好处是：

- 日常 buffer 保持 Eglot / Flymake 这套更轻的主路径
- Java、Lean 这类更适合 =lsp-mode= 的语言仍然可以显式 opt-in
- Hub 能直接反查“这个 mode 的路由是在哪个文件里声明的”

* Current Command Surface

最常用入口：

- =SPC c L=: 打开语言服务器 dispatch
- =M-x my/language-server-manager=: 打开 =*Language Server Hub*=
- =M-x my/language-server-doctor=: 打开 =*Language Server Doctor*=
- =SPC c o=: organize imports
- =SPC c R=: restart language server
- =SPC c a=: code actions
- =SPC c f=: format buffer
- =SPC c r=: rename

dispatch 里现在有四组内容：

- Views: hub / doctor / docs
- Lifecycle: ensure / restart / shutdown / log / session / config
- Edits: code actions / organize imports / format / rename
- Tuning: autoreconnect / autoshutdown / log-io / inlay hints / output max / events buffer

* Language Server Hub

命令：

- =M-x my/language-server-manager=
- 或 =SPC c L= 然后按 =h=

它展示：

- 当前 buffer 的 major mode、项目根、当前活跃 backend、workspace config
- 一组 session 级便捷调参入口
- 显式 =lsp-mode= 路由表
- 本地注册的 Eglot server 映射
- 当前 runtime knobs

Hub 里的重点不是“列个状态页面”，而是：

- 状态能看
- 命令能点
- 配置定义文件能直接跳回去

常用按键：

- =g=: refresh
- =e=: ensure 当前 buffer 的首选 backend
- =r=: restart
- =k=: shutdown
- =l=: 打开 backend log
- =s=: session / connections
- =c=: workspace config
- =o=: organize imports
- =a=: code actions
- =f=: format
- =R=: rename
- =p=: buffer problems
- =P=: project problems
- =d=: buffer diagnostics UI
- =m=: diagnostics menu
- =A=: toggle =eglot-autoreconnect=
- =S=: toggle =eglot-autoshutdown=
- =L=: toggle =lsp-log-io=
- =I=: toggle inlay hints
- =M=: 设置 =read-process-output-max=
- =E=: 设置 =eglot-events-buffer-size=
- =D=: doctor
- =O=: 打开本文档
- =RET=: 对路由/映射条目跳回定义文件

* Language Server Doctor

命令：

- =M-x my/language-server-doctor=
- 或 dispatch 里的 =D=

Doctor 主要做静态和半静态排查：

- 关键 library 是否能找到
- 本地注册的 server executable 是否存在
- 当前 buffer 的路由、active backend、workspace config
- 显式 =lsp-mode= 路由和自定义 Eglot 映射摘要
- 当前 runtime knobs

如果 LSP 状态怪异，先开 doctor，而不是先猜。

* Quick Settings

Hub 和 dispatch 里的调参入口目前都是“当前 Emacs session 生效”，不是永久写回配置。

适合场景：

- 临时打开 =lsp-log-io=
- 临时放大 =read-process-output-max=
- 临时关闭/开启 inlay hints
- 临时调大 =eglot-events-buffer-size= 看日志

如果某个值验证有效，再把它真正改回：

- =lisp/init-lsp.el=
- 或对应语言模块

* Maintenance Model

现在语言服务器相关逻辑分成三层：

1. =lisp/init-lsp.el=
   路由核心、通用 backend wrapper、全局 LSP/Flymake/company 配置。
2. =lisp/init-lsp-ops.el=
   backend-agnostic 操作命令，例如 restart / shutdown / organize imports。
3. =lisp/init-lsp-tools.el=
   Hub、Doctor、dispatch、runtime knobs、可视化维护入口。

语言专项继续放在 =lisp/lang/= 下。

* Add A New Route

如果某个语言要显式走 =lsp-mode=，用：

#+begin_src elisp
(my/register-lsp-mode-preference 'some-mode 'some-lsp-feature)
#+end_src

说明：

- 第一个参数是 major mode
- 第二个参数是这个 mode 依赖的额外 feature；没有就省略
- Hub 会自动记录这个声明来自哪个文件

例如当前已有：

- =java-mode= / =java-ts-mode= -> =lsp-java=
- =lean4-mode= -> =lsp-mode=

* Add A Custom Eglot Mapping

不要再直接散落：

#+begin_src elisp
(add-to-list 'eglot-server-programs ...)
#+end_src

现在统一用：

#+begin_src elisp
(my/register-eglot-server-program
 '(some-mode another-mode)
 '("some-language-server" "--stdio")
 :label "some-language-server"
 :executables '("some-language-server")
 :note "Short maintenance note.")
#+end_src

这样做的价值：

- Hub 能看到 modes -> server 的映射
- Doctor 能检查 executable
- 可以直接跳回配置定义文件

* Pattern For Other Modules

如果后面要给 project / shell / org / diagnostics 也补这种“陈列展示 + 管理 + 调参”界面，建议照这个结构做：

1. 主模块保留核心行为，不把维护入口越堆越大。
2. 单独拆一个 =*-tools.el= 文件。
3. 提供：
   - dispatch
   - hub
   - doctor
   - 少量安全、可回滚的 session 级调参入口
4. 主模块暴露少量 metadata / hook，让 hub 能看到“配置是怎么声明出来的”。
5. 单独补一份 workflow 文档，不把维护细节塞进 README 主叙事。

当前语言服务器这一套已经把这种模式跑通了，可以继续复用。

* Troubleshooting

如果某个 buffer 的语言服务器不工作，优先按这个顺序查：

1. 打开 =Language Server Doctor=
2. 看当前 buffer 的 =route policy= 和 =active backend=
3. 看是否命中了显式 =lsp-mode= 路由
4. 看对应 feature / executable 是否缺失
5. 看当前 buffer 是否有本地 =eglot-workspace-configuration=
6. 需要时临时打开 =lsp-log-io= 或调大 =eglot-events-buffer-size=
7. 再去开 log / session / problems / diagnostics UI

别一上来就改代码或重装 package。
